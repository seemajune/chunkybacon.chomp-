Tips for Manually Adding Files to your Gem's Library

11.27.14

While working on my most recent group project, a Rails app that requires integrating the Stanford Core Natural Language Processing library, I ran into a few issues:
  • In order to get full functionality, we had to download the StanfordCoreNLP pipeline, which does not come pre-packaged with the gem.
  • In attempt to fix this issue, I tried running bundle install, but to no avail.
  • This particular gem includes a lot of Javadoc and specific Java classes that require manual load to the gemfile's /bin sub-directory.

After discussing with my team members, and reading this very useful set of instructions for installing the stanford-core-nlp pipeline/specified docs I was able to manually install the files directly to my gem's library directory.

As a result, I am happy to say, I put to use a few things I've learned at Flatiron School from this experience:

  1. The bash command for finding the directory name that houses all of your gems is: gem environment
  2. To move any new files that you download into your gem's /bin or /lib subdirectory, you can do so directly from your terminal by typing mv /path/to/source /path/to/destination
  3. Note that, in order to move all files housed in a sub-directory, you should type the path/to/source followed by an asterisk* like so: mv /path/to/subdirectory-name/* /path/to/destination
  4. This can get messy when you're downloading a bunch of different files, so always check the downloaded files in case any of them need to be unzipped/unarchived, and unzip them before moving them into the gem's directory.
