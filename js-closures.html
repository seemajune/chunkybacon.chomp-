Closure Encounters...

12.15.14

Below is a quick example of a JavaScript closure using simple math.

function myClosure(num) {

var product = num * 8;

return function(evenNumber) {

product * evenNumber

};

}

// Notice I do not immeidately invoke the inner function. This is so that I can use it by calling it later.

var newMathStuff = myClosure(10);

// I'm calling the outer function and passing it an argument of 10.

// makes var product equal to 80 in this case. Now I have the value 80 available to me within the inner function that I am about to use ...

// Now, I can call the inner function, using the new variable 'newMathStuff'

// In doing so, I have CLOSED the outer function: (the part of the function that calculates num * 8)

newMathStuff(2);

// Here, I am using the inner function, that has been returned VIA the outer function,and passing it an argument of 2 (the 'evenNumber' parameter)

// In this case, the function multiplies the product variable by 2

// (in this case, that's 80 * 2, returning a value of 160.)

newMathStuff(4);

// Here, I call the inner function again, and now I get a value of 320.

// AND I can ALSO repurpose my closure by setting the outer most function equal to a new variable once again.

var anotherNewTrick = myClosure(50);

// product is now 400

anotherNewTrick(4);

// passes 4 to the inner function, returning 1600.

// BOOM! A simple and hopefully helpful example to understand why closures are cool.
